■ 概要
モニターごとのDPIスケーリング（拡大率）をレジストリ経由で設定します。
standard の dpi_api_config（即時反映）とは異なり、サインアウトまたは再起動後に反映されます。
Default プロファイルにも適用されるため、新規ユーザーにも設定が引き継がれます。

■ 設定ファイル
[dpi_list.csv]
  Enabled      : 有効フラグ（1=実行, 0=スキップ）
  HardwareID   : ディスプレイのハードウェアID（AUTO=自動検出、空欄=対話選択）
  ScalePercent : 拡大率（100 / 125 / 150 / 175 / 200）
  Description  : 説明（表示用）

[記載例]
  1,AUTO,150,プライマリディスプレイ150%
  1,CMN14D40,150,内蔵ディスプレイ
  0,AUO403D0,125,外部モニター

■ HardwareID の指定方法
・AUTO : プライマリディスプレイを自動検出
・具体的なID : デバイスマネージャーのモニターのハードウェアIDを指定
・空欄 : 実行時に対話形式で選択

■ 前提条件
・管理者権限が必要です

■ 注意事項
・変更の反映にはサインアウトまたは再起動が必要です

■ 動作の流れ
1. dpi_list.csv を読み込み
2. 対象ディスプレイと変更先のDPIを表示
3. 実行確認（Y/N）後、HKCU と Default プロファイルの両方にレジストリ書き込み
