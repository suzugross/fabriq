■ 概要
このモジュールには2つのスクリプトがあります。

[Registry Config (HKCU)]
  HKEY_CURRENT_USER にレジストリ値を設定します。
  同時に Default プロファイル（新規ユーザー用）にも適用されます。

[Registry Delete (HKCU)]
  HKEY_CURRENT_USER のレジストリ値を削除します。
  同時に Default プロファイルからも削除されます。

■ 設定ファイル
[reg_hkcu_list*.csv]（パターンに一致する全CSVが自動読み込み）
  Enabled      : 有効フラグ（1=実行, 0=スキップ）
  AdminID      : 管理番号（表示用）
  SettingTitle : 設定タイトル（表示用）
  KeyPath      : レジストリパス（フルパス）
  KeyName      : レジストリ値の名前
  Type         : 型（REG_DWORD / REG_SZ / REG_EXPAND_SZ / REG_QWORD / REG_BINARY / REG_MULTI_SZ）
  Value        : 設定する値

[記載例]
  1,1,タスクバー検索非表示,HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Search,SearchboxTaskbarMode,REG_DWORD,0

■ ファイル分割
ファイル名が reg_hkcu_list*.csv に一致すれば全て読み込まれます。
ジャンルごとにファイルを分けて管理できます。
例: reg_hkcu_list.csv, reg_hkcu_list_ui.csv, reg_hkcu_list_explorer.csv

■ Default プロファイルへの適用
HKCU の設定は自動的に C:\Users\Default\ntuser.dat にも反映されます。
これにより、新規作成されるユーザーにも同じ設定が適用されます。

■ Active Setup による新規ユーザーへの適用
Default プロファイルへの直接書き込みだけでは反映されない値がある場合の
補完手段として、Active Setup を使用した HKCU レジストリ書き込みも行う。

  動作の流れ:
    1. CSV の全有効エントリから reg add コマンドを動的生成
    2. C:\ProgramData\fabriq\apply_hkcu.ps1 として配置
    3. common.ps1 の Register-FabriqActiveSetup で HKLM に Active Setup を登録
       → HKLM\SOFTWARE\Microsoft\Active Setup\Installed Components\{fabriq-reg-hkcu-config}
    4. 新規ユーザーの初回ログオン時に Active Setup が
       cmd.exe 経由で apply_hkcu.ps1 を実行し HKCU レジストリを書き込む

  ON/OFF 制御:
    スクリプト先頭の $ENABLE_ACTIVE_SETUP 変数で一括制御。
    $true  = Active Setup 登録を行う
    $false = Active Setup をスキップ（既定。Default プロファイルのみで運用）

  注意事項:
    - Active Setup の登録には管理者権限が必要
    - 管理者権限がない場合は Warning を表示してスキップ
    - Default プロファイルへの直接書き込みとの併用（belt and suspenders）

■ 前提条件
・CSVのエンコーディングは Shift-JIS（ANSI）で保存してください

■ Startup Batch による新規ユーザーへの適用
Active Setup ではタイミングの問題で Explorer 初期化時にデフォルト値で
上書きされるケースがある。Startup Batch 方式は Explorer が完全に起動した
後に設定を適用し、Explorer を再起動することでこの問題を解消する。

  動作の流れ:
    1. CSV の全有効エントリから reg add コマンドを動的生成
    2. C:\ProgramData\fabriq\apply_hkcu.ps1 として配置
    3. C:\ProgramData\fabriq\fabriq_user_setup.ps1（ランチャー）を配置
    4. C:\Users\Default\...\Startup\FabriqUserSetup.cmd（トリガー）を配置
    5. 新規ユーザーの初回ログオン時:
       Startup → .cmd → フラグチェック（なし）→ PowerShell 起動
       → ランチャー → apply_hkcu.ps1 + apply_spi.ps1 実行 → Explorer 再起動
       → フラグ作成（%LOCALAPPDATA%\fabriq\user_setup_done.flag）
       → .cmd を Startup フォルダから自己削除
    6. 2回目以降のログオン:
       - .cmd が削除済みなら何も起動しない（最も軽量）
       - .cmd が残っている場合でもフラグを検出し即終了（PowerShell 起動なし）

  ON/OFF 制御:
    スクリプト先頭の $ENABLE_STARTUP_BATCH 変数で一括制御。
    $true  = Startup Batch 配置を行う
    $false = Startup Batch をスキップ（既定）

  注意事項:
    - 配置には管理者権限が必要
    - Active Setup との併用（belt and suspenders）を推奨
    - 初回ログオン時に Explorer が一瞬再起動される（タスクバーが消える）

■ 動作の流れ
1. reg_hkcu_list*.csv を全て読み込み、有効エントリを集約
2. 設定/削除対象を一覧表示
3. 実行確認（Y/N）後、HKCU と Default プロファイルの両方に適用
4. Active Setup 登録（$ENABLE_ACTIVE_SETUP が $true の場合）
5. Startup Batch 配置（$ENABLE_STARTUP_BATCH が $true の場合）
