■ 概要
Windowsの電源プランと各種電源設定（ディスプレイOFF、スリープ、休止状態、
電源ボタン動作、プロセッサ状態など）を一括設定します。
電源モード（パフォーマンススライダー）のオーバーレイ設定にも対応しています。

■ 設定ファイル
[power_list.csv]
  Enabled       : 有効フラグ（1=実行, 0=スキップ）
  ProfileName   : プロファイル名（表示用）
  Description   : 説明（表示用）
  PowerPlan     : 電源プラン（BALANCED / HIGH_PERFORMANCE / POWER_SAVER）
  PowerMode     : 電源モード（BEST_PERFORMANCE / BALANCED / BEST_EFFICIENCY）

  ※ PowerPlan と PowerMode の違い：
    PowerPlan  = ベースとなる電源プラン（powercfg /setactive で設定）
    PowerMode  = 電源プランの上に被せるオーバーレイ（powercfg /overlaysetactive で設定）
                 Windows 11: 設定 > システム > 電源とバッテリー > 電源モード
                 Windows 10: タスクバーのバッテリーアイコンのスライダー

    PowerMode の設定値：
      BEST_PERFORMANCE  = 最適なパフォーマンス
      BALANCED          = バランス（オーバーレイ解除）
      BEST_EFFICIENCY   = 最適な電力効率
      空欄 or -         = 変更しない

  以下は分単位で指定（0=無効）、AC=電源接続時、Battery=バッテリー時：
  Display_TurnOff_AC / Display_TurnOff_Battery : ディスプレイOFFまでの時間
  Sleep_After_AC / Sleep_After_Battery         : スリープまでの時間
  Hibernate_After_AC / Hibernate_After_Battery : 休止状態までの時間
  HardDisk_TurnOff_AC / HardDisk_TurnOff_Battery : HDDオフまでの時間

  以下はアクション名で指定：
  PowerButton_AC / PowerButton_Battery   : 電源ボタン動作
  SleepButton_AC / SleepButton_Battery   : スリープボタン動作
  LidClose_AC / LidClose_Battery         : カバーを閉じたとき

  アクション値: NOTHING / SLEEP / HIBERNATE / SHUTDOWN

  プロセッサ状態（%）：
  Processor_MinState_AC / Processor_MinState_Battery : 最小プロセッサ状態
  Processor_MaxState_AC / Processor_MaxState_Battery : 最大プロセッサ状態

[記載例]
  1,バランス,標準設定,BALANCED,,10,5,15,10,30,20,SLEEP,SLEEP,SLEEP,SLEEP,SLEEP,SLEEP,20,10,5,5,100,100
  0,高性能,パフォーマンス優先,HIGH_PERFORMANCE,BEST_PERFORMANCE,30,10,0,15,0,30,...

■ 前提条件
・管理者権限が必要です

■ 動作の流れ
1. power_list.csv を読み込み、設定内容を表示
2. 実行確認（Y/N）
3. powercfg コマンドで電源プランと電源モードオーバーレイを適用
4. 各設定値（ディスプレイ、スリープ、ボタン動作等）を適用

■ ボタン・カバー設定について
電源ボタン、スリープボタン、カバーを閉じたときの動作については、
powrprof.dll の Win32 API（PowerWriteACValueIndex / PowerWriteDCValueIndex）を
使用して設定します。これはコントロールパネルUIと同じコードパスであり、
powercfg.exe が機能しない一部のOEM PC（HP等）でも正しく動作します。

※ HP製PCにおける既知の表示問題：
  スクリプトで設定を適用した後、コントロールパネルの
  「カバーを閉じたときの動作」の表示が更新されない場合があります。
  これはHP固有のサービスによる表示上の問題であり、
  実際の動作およびレジストリ値（powercfg /query で確認可能）は
  正しく設定されています。
