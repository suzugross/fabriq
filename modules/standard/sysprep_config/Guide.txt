■ 概要
Sysprep の応答ファイル（unattend.xml）と初回起動スクリプト（SetupComplete.cmd）を
CSV 定義から動的に生成・配置し、Sysprep を実行します。
二段階の確認（ファイル生成 → Sysprep 実行）で安全に操作できます。

■ 設定ファイル

[sysprep_list.csv]（Sysprep 実行設定 / 単一行）
  Enabled     : 有効フラグ（1=実行, 0=スキップ）
  SysprepExe  : sysprep.exe のパス
  Mode        : 動作モード（oobe / audit）
  Shutdown    : 終了動作（shutdown / reboot / quit）
  Description : 説明（表示用）
  Segment     : セグメント（空欄 = 全対象）

  [記載例]
  1,C:\Windows\System32\Sysprep\sysprep.exe,oobe,shutdown,Sysprep実行設定,

  ※ quit を指定した場合、Sysprep 完了後にシャットダウンせず終了コードを返します。
    テスト時に便利です。

[unattend_list.csv]（応答ファイル設定 / キー・バリュー形式）
  Enabled     : 有効フラグ（1=有効, 0=無効）
  SettingName : 設定名（下記一覧参照）
  Value       : 設定値
  Description : 説明（表示用）
  Segment     : セグメント（空欄 = 全対象）

  [対応する設定名]
  --- specialize パス ---
  ComputerName        : コンピュータ名（* = 自動生成）
  CopyProfile         : プロファイルコピー（true / false）

  --- oobeSystem パス（OOBE スキップ設定） ---
  HideEULAPage              : EULA 画面のスキップ（true / false）
  ProtectYourPC              : プライバシー設定（1=推奨 / 3=送信しない）
  HideWirelessSetupInOOBE   : Wi-Fi 設定画面のスキップ（true / false）
  HideOnlineAccountScreens  : オンラインアカウント画面のスキップ（true / false）
  HideOEMRegistrationScreen : OEM 登録画面のスキップ（true / false）

  --- oobeSystem パス（アカウント設定） ---
  TestUserName        : テストユーザー名（OOBE で自動作成される LocalAccount）
  EnableAdministrator : Administrator アカウントの有効化（true / false）
  AdminPassword       : Administrator パスワード（空 = パスワードなし）

  [記載例]
  1,ComputerName,*,コンピュータ名（* = 自動生成）,
  1,CopyProfile,true,プロファイルコピー,
  1,TestUserName,Test-User,テストユーザー名,
  0,EnableAdministrator,true,Administrator アカウントの有効化,
  0,AdminPassword,,Administrator パスワード,

[setupcomplete_list.csv]（SetupComplete.cmd アクション定義）
  Enabled     : 有効フラグ（1=実行, 0=スキップ）
  Order       : 実行順序（昇順）
  ActionType  : アクション種別（DeleteUser / CopyFile / Command）
  Target      : 対象（ユーザー名 / ファイル名 / コマンド文字列）
  Destination : コピー先パス（CopyFile 時のみ使用）
  Description : 説明（表示用）
  Segment     : セグメント（空欄 = 全対象）

  [ActionType 詳細]
  DeleteUser : net user "Target" /delete を実行
  CopyFile   : source/ 内の Target を Destination へ xcopy
  Command    : Target の文字列をそのままコマンドとして実行

  [記載例]
  0,5,Command,net user Administrator /active:yes,,Administrator有効化,
  1,10,DeleteUser,Test-User,,テストユーザー削除,
  1,20,CopyFile,LayoutModification.xml,C:\Users\Default\AppData\Local\Microsoft\Windows\Shell\,タスクバー配置,
  1,30,Command,Dism /online /Import-DefaultAppAssociations:...,,既定アプリ設定,

■ source/ ディレクトリ
CopyFile アクションで使用するファイルやフォルダを配置してください。
SetupComplete.cmd 実行時に C:\Windows\Setup\Scripts\source\ へステージングされ、
そこから各宛先へ xcopy されます。

  配置例:
  source/
    LayoutModification.xml     （タスクバー設定）
    AppAssoc.xml               （既定アプリ関連付け）
    Microsoft.WindowsNotepad_8wekyb3d8bbwe/  （メモ帳設定フォルダ）

■ CSV ファイルのエンコーディング
すべての CSV ファイルは UTF-8 BOM（BOM 付き UTF-8）で保存してください。
BOM がない場合、日本語 Windows ではシステム既定の Shift-JIS として読み込まれ、
日本語の Description が文字化けします。

  確認方法（PowerShell）:
  $bytes = [System.IO.File]::ReadAllBytes("ファイル.csv")
  $bytes[0..2]  # → 239, 187, 191 (= EF BB BF) であれば BOM 付き

  再エンコード方法:
  $content = [System.IO.File]::ReadAllText("ファイル.csv", [System.Text.UTF8Encoding]::new($true))
  [System.IO.File]::WriteAllText("ファイル.csv", $content, [System.Text.UTF8Encoding]::new($true))

■ OOBE スキップ設定に関する注意
unattend_list.csv で TestUserName を指定すると、unattend.xml に <LocalAccount> ブロックが
生成されます。Windows の仕様上、<LocalAccount> が存在する場合、アカウント関連の OOBE 画面は
個別のスキップ設定に関わらず自動的にスキップされます。

  影響を受ける設定:
  - HideOnlineAccountScreens
  - HideOEMRegistrationScreen

  これらの設定を false にしても、TestUserName が有効であれば画面は表示されません。
  OOBE 画面を表示したい場合は、TestUserName を無効（Enabled=0）にしてください。

■ Administrator アカウントの有効化
unattend.xml の <AdministratorPassword> はパスワード設定のみで、
アカウントの有効化（アクティベーション）は行いません。

Administrator を有効化するには、setupcomplete_list.csv で以下の行を有効にしてください:

  0,5,Command,net user Administrator /active:yes,,Administrator有効化,
  ↑ Enabled を 1 に変更

パスワードを設定する場合は、unattend_list.csv の EnableAdministrator と AdminPassword を
それぞれ有効にしてください。

■ 動作の流れ
1. sysprep_list.csv / unattend_list.csv / setupcomplete_list.csv を読み込み
2. 前提条件チェック（sysprep.exe 存在、source/ 存在、デプロイ先作成）
3. 設定内容の一覧表示（Sysprep 設定 / Unattend 設定 / SetupComplete アクション）
4. 第 1 確認: ファイル生成・配置の実行確認（Y/N）
5. unattend.xml と SetupComplete.cmd を動的生成し、デプロイ
   - source/ 内のファイルを C:\Windows\Setup\Scripts\source\ へステージング
   - unattend.xml → C:\Windows\System32\Sysprep\unattend.xml
   - SetupComplete.cmd → C:\Windows\Setup\Scripts\SetupComplete.cmd
6. 第 2 確認: Sysprep 実行の最終確認（Y/N）
   ※ ここで N を選択した場合、ファイル配置済みで Success を返します
7. sysprep.exe を実行（/generalize /oobe /shutdown /unattend:...）

■ 前提条件
- 管理者権限が必要です
- Sysprep 実行後は PC がシャットダウンまたは再起動されます（不可逆）
- 既存の unattend.xml / SetupComplete.cmd は上書きされます

■ トラブルシューティング
- Sysprep ログ: C:\Windows\System32\Sysprep\Panther\setuperr.log
- SetupComplete ログ: C:\Windows\Setup\Scripts\SetupComplete.log
- 生成された XML の確認: C:\Windows\System32\Sysprep\unattend.xml
- 生成された CMD の確認: C:\Windows\Setup\Scripts\SetupComplete.cmd
